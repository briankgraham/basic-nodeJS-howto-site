Setting up Node

Installation


First things first: download Node via node.js.org. Once installed, you can run your JavaScipt within the terminal/command-line instead of the browser. If you have a Mac, this link can provide you with a nice walkthrough as well as show you how to install extra Node packages.

After Node is installed on your computer, you can type "node" into the command line to see if all went well:

node

>console.log('hello node');

hello node

>

There are a whole lot of tutorials for Node online so if you aren't too familiar with it you can explore much more past the how-to guide here.

What's Under the Hood?


I'm glad you asked. Normally when running JS and communicating to the server via JSON you are constantly setting up connections which over time can really bog your web site/app down and even worse, take up tons of system memory.

But Node aims to change that by allowing you to use a single thread to give the ability to run tons of connections at the same time (like thousands!).

They do this via the V8 Engine and use of non-blocking code, which we will discuss in the next section.
