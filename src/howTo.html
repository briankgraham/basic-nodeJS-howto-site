Streaming with node.JS

   by Brian Graham

About this Guide


This is a little how-to-guide on using Node.JS for web applications that require real-time data.

The guide won't teach you everything you need to know about Node, but we will go through a brief setup and a brief introduction into what Node is. Most importantly, we will go through some basics of create non-blocking code and making basic requests.

Why Use Node.js?


Well, it's not a framework...so it won't be replacing those anytime soon. However, it does use JavaScript and it was built on top of Google's V8 Engine. V8 was built mostly via C and in general: it's fast. Node is also event driven (just like JS), but its biggest feature is the ability to initiate data communication on the client-side as well as the server-side.

Think of it as an awesome 2-way portal with the potential for lots of flexibility and speed. In applications where you are reading and writing data back and forth, it can be very useful to increase speed and usability for the end-user. Also, it uses non-blocking code to allow fast data transfers, which we will discuss in another section.

So in general, Node focuses on a particular need versus being something that redefines everything you do with Web Development.

The idea behind it was to create something open-source and allow for speed that will really excite the end-user and beyond that, create a whole new world of possibilities for client-server interaction. The client could send multiple requests and the server can process all of them in parallel (You could think of it like Google Hangouts).

A great example of this would be a chat application, something that would require an 2-way open socket to allow for back and forth communication instantaneously. And not just an open socket, but a socket that can be shared amongst the entire group of people in the chat.
