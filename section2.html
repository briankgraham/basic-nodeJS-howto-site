Section 2:

Non-Blocking Code


If you are fairly familiar with Node or JavaScript, you're probably familiar with non-blocking forms of code. Personally, this is kind of eye-opening because most of my code in the past has been "blocking" in nature.

For example, take a look at this code:

var fs = require("fs");

var whatsHere = fs.readFileSync('/etc/passwrd');

console.log(whatsHere);

console.log('do more things');

So what this code does is set a variable to the readFileSync and it gets called via console.log. The file is read all the way through then the second console.log call begins.

Now check out this code:

var fs = require('fs');

fs.readFile('index.html', function(err, whatsHere){
  console.log(whatsHere);

});

The code above takes in a function attached to variable receivedIt and it's called via fs.readFile. The nice thing about this code is if you called another function afterwards, you could do it faster. Why? When receivedIt is called, it starts reading the file but immediately drops down to call the next function. Then when the file read is complete, the file data is printed to the screen.

What's great about the last code section is that it doesn't stop the program just to read a file, which is what the first section of code does. Let's say you were trying to read multiple files. You could read files concurrently with the second code but only one at a time with the first code. In summation, blocking slow, non-blocking fast.
